# dingtalk-ai-table 使用说明

## 查询限制与推荐策略

钉钉 AI 表格更适合做业务协作表格和结果展示，不适合直接当数据库做大规模、高频、复杂组合查询。

### 1. 组合查询限制

以下组合在实际使用中不可依赖：

- 排序 + 过滤 + 翻页
- 排序 + 翻页
- 过滤 + 翻页

主要问题：

- 翻页过程中数据顺序可能变化
- 表格有人编辑或 Agent 回写后，容易出现页漂移
- 可能导致重复读取、漏读取、查询结果不稳定

### 2. 过滤查询推荐方案

如果业务需要过滤查询，不要使用“过滤 + 翻页”连续拉取。

过滤条件中使用的是字段 ID，不是字段名称。Agent 在构造过滤条件前，应先读取字段列表，找到目标字段名称对应的字段 ID，再使用字段 ID 进行过滤。

示例：

```text
用户要按“商品ID”过滤

正确流程：
1. 先读取字段列表
2. 找到字段名 = 商品ID 的字段
3. 取得该字段的 fieldId
4. 使用 fieldId 构造过滤条件
```

不要直接把 `商品ID` 当成过滤字段传入。

推荐流程：

1. 使用过滤条件查询第一页
2. 处理当前返回的数据
3. 回写一个辅助标记字段
4. 下一次继续查询“未标记”的数据

示例：

```text
查询条件：
日期 = 2026-05-01
处理标记 为空

处理后回写：
处理标记 = batch_20260501_001
```

如果表里没有可用的回写字段，Agent 可以自动新增一个辅助字段，例如：

- `处理标记`
- `查询标记`
- `同步标记`
- `AI处理标记`

字段只用于避免重复读取和模拟稳定分页，不应承载业务含义。

### 3. 排序查询推荐方案

如果业务需要排序，也不要使用“排序 + 翻页”连续拉取。

推荐做法与过滤查询一致：

1. 只读取当前排序结果的第一页
2. 处理当前批次
3. 回写辅助标记字段
4. 下次查询时排除已标记数据

示例：

```text
查询条件：
处理标记 为空

排序条件：
创建时间 升序

处理后回写：
处理标记 = batch_20260501_001
```

注意：排序只用于确定当前批次的优先级，不要依赖排序 + 翻页完成全量遍历。

### 4. 大表查询建议

对于大表，不建议全表扫描后再本地过滤。

优先策略：

- 先用服务端过滤缩小范围
- 每次只处理第一页
- 通过辅助标记字段推进下一批
- 处理结果及时回写
- 统计分析类任务优先放到本地 Python / pandas / sqlite / DuckDB 中完成

### 5. 时间过滤能力不足

当前更适合按日期过滤。

支持：

```text
2026-05-01 ✅
```

不支持或不可依赖：

```text
2026-05-01 01:00 ~ 2026-05-01 02:00 ❌
```

推荐做法：

- 表格侧按日期过滤
- 拉取当天数据后，在本地代码中按小时、分钟、秒继续筛选
- 如果需要高频时间段查询，建议增加辅助日期字段或本地缓存

### 6. 图片字段性能较差

图片 / 附件字段通常会显著拖慢查询速度。

可能原因：

- 附件元数据较大
- 需要生成图片访问地址
- 需要额外权限校验
- 返回 JSON 体积变大

推荐策略：

- 如果用户没有明确要求图片字段，默认只返回非图片字段
- 查询前先读取字段信息，排除图片 / 附件类型字段
- 需要图片时，再单独查询图片字段或按记录补查

示例：

```text
用户需求：查询今天的销售记录

推荐返回字段：
店铺、日期、销售额、花费、ROI、备注

默认排除字段：
截图、图片、附件、凭证
```

### 7. Agent 默认查询规则

Agent 执行查询时建议遵守以下规则：

1. 不使用“排序 + 过滤 + 翻页”组合
2. 不使用“排序 + 翻页”组合
3. 不使用“过滤 + 翻页”组合
4. 构造过滤条件前，先通过字段列表把字段名称转换为字段 ID
5. 需要批量处理时，使用“第一页 + 回写标记”的方式推进
6. 没有回写字段时，自动新增一个辅助标记字段
7. 用户没有要求图片时，默认排除图片 / 附件字段
8. 时间范围精确到小时、分钟、秒时，先按天查，再本地筛选
9. 大规模统计分析不要依赖 AI 表格实时查询，优先本地计算后回写结果

# 示例数据文件

## fields.json - 批量新增字段示例

```json
[
  {
    "fieldName": "任务名",
    "type": "text"
  },
  {
    "fieldName": "优先级",
    "type": "singleSelect",
    "config": {
      "options": [
        {"name": "高"},
        {"name": "中"},
        {"name": "低"}
      ]
    }
  },
  {
    "fieldName": "截止日期",
    "type": "date"
  },
  {
    "fieldName": "负责人",
    "type": "user",
    "config": {
      "multiple": false
    }
  },
  {
    "fieldName": "进度",
    "type": "progress"
  }
]
```

## data.csv - 批量导入记录示例

```csv
fld_name,fld_age,fld_status,fld_salary
张三,25,进行中,15000
李四,30,已完成,18000
王五,28,进行中,16000
```

## data.json - JSON 格式导入示例

```json
[
  {
    "cells": {
      "fld_name": "张三",
      "fld_age": 25,
      "fld_status": "进行中",
      "fld_salary": 15000
    }
  },
  {
    "cells": {
      "fld_name": "李四",
      "fld_age": 30,
      "fld_status": "已完成",
      "fld_salary": 18000
    }
  }
]
```

## 字段类型参考

| 类型 | 说明 | 示例 |
|------|------|------|
| `text` | 文本 | `"张三"` |
| `number` | 数字 | `25` |
| `singleSelect` | 单选 | `{"name":"高"}` |
| `multipleSelect` | 多选 | `[{"name":"高"},{"name":"紧急"}]` |
| `date` | 日期 | `"2026-03-31"` |
| `user` | 用户 | `{"id":"user_xxx"}` |
| `checkbox` | 复选框 | `true` |
| `attachment` | 附件 | `[{"fileId":"file_xxx"}]` |
| `url` | 链接 | `{"text":"官网","link":"https://..."}` |
| `richText` | 富文本 | `{"markdown":"**加粗**"}` |
